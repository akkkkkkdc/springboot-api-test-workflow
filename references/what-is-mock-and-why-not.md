# mock ≠ 真测试：为什么 mock 自测骗自己

## 什么是 mock

`mock` = 假服务器。你**自己写代码**假装是后端，返回你想让它返回的数据。

```
真流程：    Newman → 你公司后端 → MySQL → 真数据
我前面做的：Newman → 我写的 50 行 Node 脚本（假装是后端）→ 假数据
```

## 为什么 mock 自测 = 玩具自我打勾

```
断言"业务码 = 0"  →  mock 自己写的返回值就是 0  → 当然过
断言"文件 > 1KB"  →  mock 自己返了 1760 字节      → 当然过
```

相当于你考了 100 分，因为**自己出题 + 自己写答案 + 自己改卷**。

## mock 自测能验证什么

| 能验证 | 不能验证 |
|---|---|
| ✅ 工具链通不通 | ❌ 后端 SQL 写错没 |
| ✅ Newman 报告生成正不正常 | ❌ 字段名跟清单一不一致 |
| ✅ HTML 模板渲染对不对 | ❌ 权限控制工作没工作 |
| ❌ **你的接口过没通过** | ❌ 导出 Excel 真不真 |

## 什么时候可以 mock

- 写工具时**先 mock 自测**——验证脚本本身能跑
- 然后**真打服务**——验证接口本身
- 两步不能省

## 怎么从 mock 切到真测

3 件事：
1. 服务**跑起来**（不是 skill 的事）
2. 知道服务地址（`http://localhost:9123`）
3. 知道怎么登录（要么账号密码，要么直接给 token）

有这 3 件事才能真测。**没服务 = 没真测 = 没报告**。
